# GUI 가이드

## 실행
```bat
python gui_bridge.py
```

## 기본 흐름
1. 심볼 선택
2. 기간 선택
3. 전략 선택
4. 필요 시 JSON 파라미터 조정
5. 전략 테스트 실행
6. 결과 확인 후 엔진 적용

## 지원 모드
- 단일 전략 테스트
- 전체 전략 일괄 테스트
- 기간별 전체 전략 일괄 테스트

## 버튼별 결과 해석
- `1) strategy.py로 테스트`: 현재 선택 전략의 성능 지표 확인
- `1-A) 전체 전략 일괄 테스트`: 같은 기간에서 전략 간 상대 비교
- `1-B) 전체 전략×기간 일괄 테스트`: 기간별 강건성까지 포함한 순위 확인
- `2) ultra_quant.py에 즉시 적용`: 마지막 테스트의 선택 전략을 실행 엔진으로 전달

## 결과 확인 우선순위
1. 최근 구간 점수(3~6개월)
2. MDD와 Sharpe 균형
3. 거래 횟수(너무 적으면 과최적화 의심)
4. 기간별 일관성

## `symbols.json` 사용
루트의 `symbols.json`로 GUI 심볼 목록을 관리합니다.

예시:
```json
{
  "symbols": ["SPY", "QQQ", "AAPL", "BTC-USD", "EURUSD=X"]
}
```

## 자주 발생하는 문제
- 심볼 데이터 조회 실패: 심볼 표기(`BTC-USD`, `EURUSD=X`) 확인
- 결과가 비정상적으로 좋음: 거래 횟수, 수수료 반영, OOS 결과 재확인
- 한글 출력 깨짐: PowerShell에서 `chcp 65001` 적용
