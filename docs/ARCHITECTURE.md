# 아키텍처 개요

## 엔진 계층
- 실행 엔진: `ultra_quant.py`
- 전략/연구 계층: `strategy.py`
- UI 브리지: `gui_bridge.py`

## 데이터 흐름
1. 시장 데이터 수집/적재
2. 전략 시그널 생성
3. 백테스트 실행 및 성능 계산
4. OOS/리스크 분석
5. GUI 또는 실행 엔진으로 적용

## 성능
- Numba JIT 기반 계산 가속
- Ray 기반 병렬/분산 처리 확장
- 파라미터 스윕 및 반복 백테스트 최적화

## 리스크 분석
- Monte Carlo 기반 분포 시뮬레이션
- VaR/CVaR 중심 손실 구간 점검
- Walk-Forward 기반 과최적화 완화

## 보안
- API 키 암호화 저장
- 2FA(TOTP) 기반 접근 보호
- 운영 환경에서 평문 비밀값 저장 지양

## 멀티마켓
- 미국 주식, 국내 데이터, 암호화폐, FX 확장 구조
- 브로커/거래소 어댑터 패턴으로 연결 계층 분리

## 운영 원칙
- 연구(`strategy.py`)와 실행(`ultra_quant.py`)을 분리해 책임 경계 유지
- 실행 전 검증 기준(OOS/리스크/비용 모델)을 문서로 고정
- 신규 전략 추가 시 동일한 검증 절차로 비교 가능성 확보
